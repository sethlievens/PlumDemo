# Database Schema

## Overview

This project models a retail grocery data warehouse using a **star schema** optimized for analytical workloads. The design separates descriptive dimensions from transactional fact tables while supporting inventory optimization, demand forecasting, phantom inventory detection, and replenishment recommendations.

---

# Dimension Tables

Dimension tables provide descriptive attributes used for filtering, grouping, and reporting.

| Table | Purpose |
|--------|---------|
| **dim_store** | Store information including location, store type, cluster, and peer group. |
| **dim_item** | Product catalog containing pricing, perishability, shelf life, and replenishment attributes. |
| **dim_department** | Product department hierarchy. |
| **dim_date** | Calendar dimension with holidays, weekends, and seasonal attributes. |
| **dim_vendor** | Vendor information including lead times and delivery schedules. |
| **dim_scenario** | Simulation scenarios used to compare historical, baseline, and optimized inventory strategies. |

---

# Fact Tables

Fact tables store measurable business events.

| Table | Grain | Description |
|--------|-------|-------------|
| **fact_sales** | Store × Item × Day | Daily sales transactions with revenue, cost, and promotional flags. |
| **fact_inventory_snap** | Store × Item × Day | End-of-day inventory snapshots including inventory value and days of supply. |
| **fact_receipts** | Purchase Order | Vendor deliveries and replenishment receipts. |
| **fact_waste** | Waste Event | Inventory discarded due to expiration or spoilage. |
| **fact_lost_sales** | Store × Item × Day | Simulated demand used to measure stockout impact and lost revenue. |

---

# Operational Tables

These tables support the optimization and simulation workflow.

| Table | Purpose |
|--------|---------|
| **par_levels** | Stores recommended inventory levels generated by the replenishment algorithm. |
| **phantom_events** | Ground-truth record of injected phantom inventory events used for validation. |
| **detection_log** | Results produced by the phantom inventory detection algorithm, including confidence scores and estimated lost sales. |

---

# Simulation Scenarios

The warehouse supports multiple inventory planning scenarios without modifying historical data.

Typical scenarios include:

- Historical
- Baseline Forecast
- Recommended Par Levels
- Aggressive Inventory Strategy

This design allows multiple optimization strategies to be compared side-by-side using the same underlying transactional history.

---

# Design Goals

- Star schema optimized for SQL Server analytical workloads
- Clear separation between dimensions and transactional facts
- Support for historical reporting and forward simulation
- Inventory optimization using stored procedures
- Phantom inventory detection with measurable ground truth
- Demonstration of execution plan analysis and SQL Server performance tuning
